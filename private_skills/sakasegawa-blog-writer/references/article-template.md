# 記事構成テンプレート

## フロントマター

```yaml
---
title: "{タイトル}"
description: "{1-2文の説明}"
date: YYYY-MM-DD
tags: [{タグ1}, {タグ2}]
author: 逆瀬川ちゃん
---
```

## 章間接続の例

以下は実際の記事から抽出した接続パターン。

### パターン1: 「さて」で前章を受けて展開
前章末: 「...これがContext CachingやPrompt CachingやPrefix Cachingの基本原理です。」
次章頭: 「さて、キャッシュ化はされたのですが、うまく使われなきゃいけません。」

### パターン2: 「ここで」で振り返りを促す
「ここで現代のAI Agent (Coding Agentなど) のContext構造を振り返っていきましょう。」

### パターン3: 「このように」で前章を要約して次へ
「さて、このように積み重なるContextを見たときに、AI AgentにおけるContext Engineeringはより具体的になります。」

### パターン4: 結論から入って理由を後述
「めっちゃ安いからです (結論)」→ 詳細な価格表と計算が続く

### パターン5: 前章の疑問を受けて回答
前章末で暗に疑問を残す → 次章頭でその疑問に答える形で始める
